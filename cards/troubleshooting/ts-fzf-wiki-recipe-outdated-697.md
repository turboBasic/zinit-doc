## Summary

Loading fzf shell integration scripts directly from the fzf git repository (e.g. `key-bindings.zsh`, `completion.zsh`) can break when the fzf binary is updated to a newer version, because the scripts from `master` may not be compatible with the installed binary.

## Symptom

fzf keybindings or completions stop working or produce errors after running `zinit update` to get a new fzf binary, even though the scripts are also up-to-date.

## Cause

The shell scripts (`key-bindings.zsh`, `completion.zsh`) in the fzf source repository track the latest fzf development and may not be compatible with a specific released binary version. The wiki recipe pointed users at `master` branch scripts, which can diverge from the release being installed.

## Fix / Workaround

Use the official fzf shell integration approach — let fzf itself generate the correct scripts for the installed version:

```zsh
zinit ice from'gh-r' as'command'
zinit light junegunn/fzf

# In .zshrc, after fzf is available:
source <(fzf --zsh)
```

This always uses scripts generated by the exact installed binary version, guaranteeing compatibility.

## Caveats

`fzf --zsh` requires fzf 0.48.0 or newer. For older versions, use the matching scripts from the same tag as the installed binary.
